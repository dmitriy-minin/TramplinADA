# Аудит готовности к 2 этапу «IT‑Планета 2026» (на 26.03.2026)

## Краткий вывод

Проект уже покрывает значимую часть базового функционала (роли, регистрация/авторизация, CRUD возможностей, отклики, избранное, карта, базовая админка), но в текущем состоянии не проходит как минимум один критический технический порог: фронтенд не собирается из-за отсутствующего `@/lib/api`.

При этом есть и функциональные разрывы относительно задания: нет корректного потока модерации PENDING, нет контактной сети (друзья/рекомендации), неполное покрытие ЛК работодателя и куратора, нет формализованной демонстрации «авторских решений и их обоснования».

## Что уже хорошо реализовано

1. **RBAC и базовая безопасность**
   - Роли `soiskatel / employer / curator` есть в модели пользователей.
   - JWT access/refresh и проверка ролей в зависимостях присутствуют.

2. **Основной домен возможностей**
   - Есть сущности `Opportunity`, `Tag`, связи, фильтрация, карта, просмотр карточки.
   - Есть модерация возможности куратором (approve/reject endpoint).

3. **Пользовательские сценарии**
   - Регистрация/логин, профиль соискателя, отклики, избранное.
   - Создание возможностей работодателем.

4. **Кураторские инструменты (частично)**
   - Просмотр/деактивация пользователей.
   - Верификация компаний.
   - Управление тегами.

## Критические блокеры (исправить в первую очередь)

1. **Фронтенд не проходит type-check/build**
   - Отсутствует модуль `@/lib/api`, который импортируется в большинстве страниц.
   - Есть дополнительные ошибки типизации и неиспользуемых импортов.
   - Без исправления нельзя стабильно собрать демо и видеоролик.

2. **Поток модерации фактически сломан в UI/API связке**
   - UI админки запрашивает pending через `status=pending`.
   - Backend `GET /opportunities` сейчас возвращает только `ACTIVE` и не принимает фильтр `status`.
   - В результате куратор не видит очередь модерации через текущий интерфейс.

3. **Отсутствует часть обязательного функционала из задания**
   - Нет реализованных API/экранов для профессиональных контактов (friend/networking).
   - Нет механики рекомендаций вакансий/мероприятий между соискателями.

## Риски по критериям оценки

1. **Реализация базового функционала (0–60)**: риск средний из-за сломанной сборки и модерации.
2. **ЛК соискателя (0–40)**: средний, есть база, но нет контактов/рекомендаций.
3. **ЛК работодателя (0–50)**: средний/высокий, не закрыт сценарий работы с откликами в UI.
4. **ЛК куратора (0–60)**: средний/высокий, модерация неполная.
5. **Карточки возможностей (0–50)**: средний, база есть, но не все поля задания и модальная карта.
6. **Авторские решения и обоснование (0–100)**: высокий риск при отсутствии явной документации.
7. **Качество реализации и дизайн (0–80)**: визуально хорошо, но технический долг (type-check/build) снижает оценку.
8. **Презентация и видео (0–60)**: риск зависит от стабилизации MVP.
